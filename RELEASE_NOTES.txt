AstroImages Release Notes

v1.6.0 (from v1.5.0)

New Features:
* Added an applicaiton installer 
    - Previously you had to download a zip file and place it someplace.
* Added display of metadata from a file in the directory called AquisitionDetails.csv and ImageMetaData.csv
    - These files are written by Tom Palmer's Session Metadata plugin for NINA
    - The csv files have the full pathname but only the filename part is used to match files in the current folder.
* Added median display format options: normalized range (0.0-1.0) or 16-bit range (0-65535).
* Added an optional histogram panel that can be shown or hidden from Options.
* Added floating image and histogram windows for multi-monitor workflows.
* Added file-list column color coding:
	- Green: custom filename keywords
	- Blue: FITS keywords
	- Purple: median/mean columns
    - Orange: Session Metadata values


Improvements:
* Improved dark mode text visibility across the application.
* Improved column sizing behavior for mixed metadata sets.
* Expanded NINA filename parsing support (including FWHM and RMSARCSEC patterns).
* Added compact astronomical coordinate display formatting coordinates from the fits header
	- RA shown as HH:MM:SS
	- Dec shown as +DD:MM:SS or -DD:MM:SS

Fixes:
* Fixed grey/unused areas when histogram is disabled.
* Fixed floating/docking layout issues and window resize behavior.
* Fixed file-list width behavior when adding/updating Session Metadata columns.
* Fixed metadata-column width calculation in floating mode so the file list can properly use available width.

Notes:
* This release consolidates usability and layout improvements introduced after v1.5.x, with emphasis on metadata readability and multi-window workflows.
